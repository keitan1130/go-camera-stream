# WebRTC P2P Camera Streamer

スマートフォン（iPhone等）のカメラ映像を、超低遅延でPC（OBSブラウザソース等）へ配信するためのWebRTC（P2P型）ストリーミングアプリです。

Go言語による軽量なシグナリングサーバー（WebSocket）と、Reactによるフロントエンドで構成されています。
映像データはサーバーを経由せず、端末間で直接（Peer-to-Peer）通信されるため、サーバーへの負荷が極めて低く、ミリ秒単位の超低遅延を実現しています。

**主な特徴:**
* **LAN内完全対応**: 外部のSTUNサーバーに依存しない設定になっているため、インターネットに繋がっていない完全なローカルネットワーク（Wi-Fi）内でも最速で動作します。
* **カメラレンズの明示的な選択**: iPhoneの超広角・広角・望遠レンズなど、特定の物理カメラをUIから指定して配信できます。
* **直感的なステータス表示**: 実際にP2Pで映像データが届いているか（配信中か、スタンバイ中か）を画面上で正確に確認できます。

## アクセスURL

サーバー起動後、以下のURLにアクセスして利用します。
※IPアドレス部分は、環境変数 `PUBLIC_HOST` の設定値や、サーバーマシンの実際のローカルIPに読み替えてください。

### 1. カメラ配信側（スマートフォン等）
映像を配信するデバイスで以下のURLを開きます。
**※必ずこちらを先に開いて、配信を開始してください。**

```text
https://[サーバーのIPアドレス]:8080/?id=1

```

*(例: `https://192.168.1.100:8080/?id=1`)*

### 2. 視聴側（OBS / PCブラウザ）

映像を受信したいPC、またはOBS Studioの「ブラウザソース」で以下のURLを開きます（`&mode=viewer` が必要です）。

```text
https://[サーバーのIPアドレス]:8080/?id=1&mode=viewer

```

*(例: `https://192.168.1.100:8080/?id=1&mode=viewer`)*

> **接続数に関する注意（P2Pの仕様）:**
> 技術的には複数人での同時接続が可能ですが、本アプリはサーバーを経由しないP2P通信を採用しているため、視聴者が増えるほどカメラ端末（スマートフォン等）のエンコード負荷とネットワーク帯域の消費が倍増します。安定した配信のため、**1〜2デバイス（自分用のOBSと確認用ブラウザなど）での視聴を推奨**します。

> **セキュリティ警告について:**
> カメラやマイクにアクセスするには **HTTPS接続** が必須です。本サーバーは起動時に自己署名証明書を自動生成するため、ブラウザで開いた際に「安全ではありません」という警告が出ます。「詳細設定」から「(安全ではない)サイトへ進む」を選択してアクセスしてください。

## 本番環境（Dockerを用いた起動）

DockerとDocker Composeがインストールされている環境であれば、最も簡単に起動できます。

### 1. 環境変数の設定（任意）

`.env.example` をコピーして `.env` ファイルを作成し、ご自身のネットワーク環境に合わせてIPアドレス等を設定してください。

```bash
cp .env.example .env

```

### 2. ビルドと起動

```bash
# バックグラウンドでビルドして起動
docker compose up -d --build

# ログを確認する場合
docker compose logs -f

```

## 開発環境（ローカル開発での起動）

コードを直接編集しながら開発を行う場合の手順です。Go (1.20以上) と Node.js (v20推奨) が必要です。

### 1. フロントエンド（React）のビルド

```bash
cd frontend
npm install
npm run build
cd ..

```

*※Goサーバーは `frontend/dist` ディレクトリの中身を静的ファイルとして配信するため、フロントエンドを変更した際は必ずビルドが必要です。*

### 2. バックエンド（Go）の起動

```bash
# 依存パッケージの整理・ダウンロード
go mod tidy

# サーバーの起動
go run *.go

```

*※デフォルトでは `localhost:8080` で起動します。ローカルネットワーク上の他の端末からアクセスしたい場合は、環境変数をつけて起動してください。*

```bash
PUBLIC_HOST=192.168.1.100 go run *.go

```

## 環境変数

`.env` または起動時のコマンドラインで以下の環境変数を設定できます。

| 変数名 | デフォルト値 | 説明 |
| --- | --- | --- |
| `APP_PORT` | `8080` | Webサーバー、WebSocket、および静的ファイルの配信ポートです。 |
| `PUBLIC_HOST` | `127.0.0.1` | 外部からアクセスする際のIPアドレス（例: `192.168.1.100`）。自己署名証明書の生成にも使用されます。 |
| `TLS_CERT_FILE` | (なし) | 正規のSSL証明書を使用する場合の証明書ファイルのパス。指定がない場合は自己署名証明書が自動生成されます。 |
| `TLS_KEY_FILE` | (なし) | 正規のSSL証明書を使用する場合の秘密鍵ファイルのパス。 |

## プロジェクト構成

* **バックエンド (Go)**
* `main.go`: サーバー起動と静的ファイルの配信設定。
* `tls.go`: TLS証明書（自己署名証明書）の生成・読み込み管理。
* `signaling.go`: WebRTCの接続を仲介するWebSocketシグナリングおよび部屋（Room）管理。


* **フロントエンド (React + Vite)**
* `frontend/src/constants.ts`: 解像度やFPS、WebRTC設定などの共通定数。
* `frontend/src/App.tsx`: モード（カメラ/視聴）の判定とルーティング。
* `frontend/src/components/CameraMode.tsx`: 送信側UIとWebRTCオファー制御、カメラデバイス管理。
* `frontend/src/components/ViewerMode.tsx`: 受信側（OBS/PC）UIとWebRTCアンサー制御。
* `frontend/src/App.css`: UIのスタイリング。


* **インフラ**
* `Dockerfile` / `docker-compose.yml`: 本番用コンテナ設定ファイル。
